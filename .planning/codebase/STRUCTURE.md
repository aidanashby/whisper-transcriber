## Directory Purposes

**`src/`:**
- Purpose: All application source code
- Contains: Python modules only; no static assets
- Key files: `main.py` (entry), `controller.py` (state), `app.py` (root window)

**`src/ui/`:**
- Purpose: All CustomTkinter and tkinter UI code
- Contains: Panel classes, widget subclasses, visual constants
- Key files: `constants.py` (all magic values), `left_panel.py`, `right_panel.py`

**`assets/`:**
- Purpose: Static assets for the built application
- Contains: `icon.ico` only (generated by `create_icon.py`)
- Generated: Yes (by `build.bat` step 2), but committed once generated

**`build/`:**
- Purpose: PyInstaller intermediate artefacts
- Generated: Yes
- Committed: No (in `.gitignore`)

**`dist/`:**
- Purpose: Final distributable output
- Generated: Yes
- Committed: No (in `.gitignore`)

## Key File Locations

**Entry Points:**
- `run.py` вЂ” PyInstaller entry point; do not add logic here
- `src/main.py` вЂ” real entry point; `main()` contains full startup sequence

**Configuration / Constants:**
- `src/ui/constants.py` вЂ” every colour, font, timing, and dimension value
- `requirements.txt` вЂ” dependency versions
- `transcriber.spec` вЂ” build configuration including CUDA DLL discovery logic

**Core Logic:**
- `src/controller.py` вЂ” all application state and inter-thread message dispatch
- `src/transcription_worker.py` вЂ” Whisper model management, batch transcription loop
- `src/audio_processor.py` вЂ” ffmpeg invocation, temp file management

**Testing:**
- No test directory detected. No test files present.

## Naming Conventions

**Files:**
- `snake_case.py` throughout (e.g., `left_panel.py`, `audio_processor.py`)
- UI files live under `src/ui/` regardless of whether they are panels or dialogs

**Classes:**
- `PascalCase` (e.g., `AppController`, `TranscriptionWorker`, `LeftPanel`, `FileRow`)

**Constants:**
- `UPPER_SNAKE_CASE` in `src/ui/constants.py` (e.g., `QUEUE_POLL_MS`, `ROW_COMPLETE_DONE`)

**Private methods/attributes:**
- Leading underscore prefix (e.g., `_poll_queue`, `_is_running`, `_on_drop`)

## Where to Add New Code

**New UI panel or dialog:**
- Implementation: `src/ui/<name>.py`
- Import in: `src/app.py` or wherever it is instantiated
- Visual constants: add to `src/ui/constants.py`; never hard-code values inline

**New controller feature (state, worker interaction):**
- Implementation: `src/controller.py`
- Worker communication: add a new queue message tuple type; handle in `_dispatch()`

**New worker capability:**
- Implementation: `src/transcription_worker.py`
- New callback: add field to `TranscriptionCallbacks` (use `Optional` with default `None`)

**New audio processing step:**
- Implementation: `src/audio_processor.py:AudioProcessor` (add static method)

**New build step:**
- Add to `build.bat`; if PyInstaller packaging changes are needed, edit `transcriber.spec`

**New dependency:**
- Add to `requirements.txt`
- If PyInstaller misses it, add to `hiddenimports` or `collect_all()` calls in `transcriber.spec`

## Special Directories

**`dist/WhisperTranscriber/_internal/`:**
- Purpose: All bundled Python packages, DLLs, Tcl/Tk data, CUDA binaries
- Generated: Yes (by PyInstaller COLLECT step)
- Committed: No
- Note: nvidia CUDA DLLs (`cublas64_12.dll`, etc.) land here via `_find_nvidia_dlls()` in spec

**`.planning/`:**
- Purpose: GSD workflow planning documents and state
- Generated: No (manually maintained)
- Committed: Yes
